Добро пожаловать в мой личный репозиторий конфигурационных файлов (**dotfiles**). Этот сетап спроектирован специально для работы на ультра-слабом железе (Low-End PC / Ноутбуки) и оптимизирован под повседневные задачи исследователя информационной безопасности и системного администрирования.

## 💻 Спецификации хост-машины (Target Hardware)
Многие считают, что современный Linux требует мощного процессора, но этот сетап доказывает обратное, работая быстро и отзывчиво на следующем железе:
- **Устройство:** Ноутбук Lenovo G50-45
- **Процессор:** AMD A4-6210 (2-4 ядра с низкой тактовой частотой)
- **Оперативная память:** 4 ГБ DDR3 (Жесткий лимит, требующий ювелирного контроля за RAM)
- **Графическое окружение:** i3wm (Тайлинговый оконный менеджер вместо тяжелых GNOME/KDE)

## 📊 Модуль 1: Системный мониторинг и Хакерский Кокпит (btop)

Основным инструментом для контроля за состоянием системы является утилита **btop**. Конфигурационный файл (`config/btop/btop.conf`) полностью переработан вручную для минимизации нагрузки на процессор и видеокарту, а также для создания эстетичного текстового интерфейса в стиле киберпанк.

### ⚙️ Особенности конфигурации btop:
1. **Экстремальная оптимизация:** Интервал обновления графиков увеличен до оптимального, чтобы сам процесс мониторинга не сжирал драгоценные циклы слабого процессора AMD A4.
2. **Текстовые ASCII-часы:** Большие информативные часы интегрированы прямо в интерфейс терминала, что исключает необходимость держать включенными тяжелые графические виджеты на рабочем столе.
3. **Компактный вывод:** Панели процессора, оперативной памяти, дисков (`root`, `home`, `swap`) и сетевых интерфейсов расположены так, чтобы вся критическая информация помещалась на одном экране без необходимости прокрутки.
4. **Контроль утечек:** Настройки btop адаптированы для мгновенного обнаружения «прожорливых» фоновых процессов, что критически важно при общем объёме RAM всего в 4 ГБ.

### 🚀 Пошаговая установка конфигурации btop

Если вы хотите развернуть точно такой же легкий и информативный дашборд на своей системе, выполните следующие шаги:

1. Установите утилиту btop через стандартный пакетный менеджер Arch Linux:
```bash
sudo pacman -S btop
```

2. Создайте системную директорию для конфигурационных файлов btop, если она ещё не существует в вашей домашней папке:
```bash
mkdir -p ~/.config/btop
```

3. Склонируйте этот репозиторий и принудительно скопируйте мой конфигурационный файл поверх стандартного:
```bash
cp config/btop/btop.conf ~/.config/btop/btop.conf
```

4. Запустите монитор из любой точки терминала и наслаждайтесь скоростью работы:
```bash
btop
```
